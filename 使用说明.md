# 国自然基金申请助手 · 使用说明

这是一个帮你打磨**国家自然科学基金申请书**的智能助手，能帮你
**诊断选题、写立项依据、理研究方案、模拟评审、润色并生成合规标注**。
不需要懂电脑，照下面三步就能用。

---

## 第一步：安装（只需做一次）

1. 找到 `scripts` 文件夹里的 `setup.ps1` 文件。
2. **右键点它 → 选择「使用 PowerShell 运行」**。
3. 看到绿色的「安装完成！」字样就好了（第一次会下载一些东西，请耐心等几分钟）。

> 如果右键没有「使用 PowerShell 运行」，可以在文件夹空白处按住 Shift 右键 →「在终端中打开」，
> 然后输入：`powershell -ExecutionPolicy Bypass -File scripts/setup.ps1` 回车。

## 第二步：填一次 API 密钥

本助手需要连接一个 AI 大模型才能工作。

1. 打开 `backend` 文件夹里的 `.env` 文件（用记事本即可）。
2. 把 `LLM_API_KEY=` 后面换成你自己的密钥。
   - 默认用的是 **DeepSeek**（便宜好用）。去 deepseek.com 注册可获得密钥。
   - 也可以换成硅基流动、OpenAI 或 Claude，文件里有注释说明怎么切换。
3. 保存关闭。

## 第三步：使用

以后每次使用，只要**双击 `启动基金助手.bat`**，浏览器会自动打开界面。
用完后，把弹出的黑色小窗口关掉就行。

---

## 五大功能怎么用

| 功能 | 你要做的 |
|---|---|
| 💡 **选题诊断** | 填上拟报项目名称（学科、科学问题、创新点选填），它以**评审专家视角**评估创新性、帮你归类研究属性（自由探索类 / 目标导向类，2024 年起的新分类），并挑出最可能被毙的三条硬伤。 |
| 📚 **立项依据** | 填研究方向（关键词建议用英文），它会**真实检索 PubMed**，据此撰写立项依据草稿（研究现状→不足→切入点）；引用都是可点开的链接，并会**自动核验有没有编造的文献**。 |
| 🗺️ **研究方案** | 写下研究构想，它帮你理成「研究目标—研究内容—关键科学问题—技术路线（带流程图）—可行性」，并提示目标与内容是否对应。 |
| 🧐 **评审模拟** | 把申请书内容粘进去（或拖入草稿），**三位不同风格的评审**给意见、打分、挑刺，最后汇总致命问题。 |
| ✍️ **润色合规** | 把文本粘进去，它润色成规范基金书面语（**只改表达、不改内容、不编造**），并生成一段**符合基金委要求的 AI 使用标注**，可据实修改后附在材料里。 |

> 🔗 **一键串联**：选题诊断 → 立项依据 → 研究方案 → 评审模拟，结果可直接带到下一步。
> 📎 每个页面都支持**拖拽上传** Word/PDF/txt，自动读取文字作为内容或参考资料。
> 💾 结果可导出 Markdown 或 Word，方便粘进官方申请书模板继续修改。

## ⚠️ 必读：关于合规

国家自然科学基金委明确规定：

- **不得直接使用 AI 生成的申请书**、进展报告、结题报告；
- AI **可以**用于跟踪研究动态、检索整理文献、语言润色等辅助环节；
- 但**必须由你本人核实** AI 生成的所有信息和参考文献的真实性、准确性；
- 不得隐瞒 AI 使用，需按规定**如实标注**。

因此：**本工具的一切产出都是草稿和建议，核心科学思想、研究方案、结论必须由你独立完成；
正文引用的每一篇文献都要你亲自核对后才能用。** 「润色合规」模块会帮你生成一段可用的标注文字。

## 让同事/同组也能用（局域网，可选）

如果你想让**同一个 Wi-Fi / 路由器下**的其他电脑或手机也能用你这台机器上的助手：

1. 右键 `scripts\serve-lan.ps1` →「使用 PowerShell 运行」（或双击 `局域网部署.bat`）。
2. 窗口里会显示一个像 `http://192.168.1.6:8766` 的网址。
3. 让别人在他们的浏览器里打开这个网址就能用了（不用在他们电脑上装任何东西）。

> ⚠️ **请注意**：开了局域网后，同网络里的人都能用你这台机器，也会**花你的 AI 额度**，请只在信得过的内网用，用完关掉窗口即可。
> 如果别人打不开，多半是 Windows 防火墙挡住了——用**管理员身份**运行那个脚本会自动放行。

## 常见问题

**问：左下角显示「正在连接本地服务」？**
答：说明后台还没起来。请确认你是通过双击 `启动基金助手.bat` 打开的，且那个黑色窗口还开着。

**问：提示「余额不足」之类的错误？**
答：你的 AI 密钥额度用完了，去对应平台充值，或在 `.env` 里换一个密钥（也可配置备用供应商自动切换）。

**问：立项依据检索不到文献？**
答：PubMed 主要覆盖生物医学领域，且检索关键词建议用**英文**。非医学领域可只用「选题诊断 / 研究方案」模块。

**问：和「科研助手」冲突吗？**
答：不冲突。本助手用 8766 端口，科研助手用 8756，可同时运行。
